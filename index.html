Roadmap

In Progress:

Pre-Render Variable Content

Before switching to a backend server, the website had to load in user generated content into template pages right after the user loads the page. There is a visible delay before that content is loaded, during which, the user can see placeholder content. Additionally, the content sometimes wouldn't load until you refreshed the page, and bots and crawlers don't load the content at all. This was not good for search engine optimization.

With a backend server now in place, this content can be loaded into a template before the page is served to the user. Additionally, the backend server allows for cleaner, more SEO-friendly URLs that don't rely on hacky 404 pages or weird URL structures.

Furthermore, this allows to load content like navbar & footer, as well as appearance settings before the page is served to the user.

URLs like /article?id=some-id will be moving to URLs like /article/some-id, and dynamic metadata, like article previews, will now be loaded into link embeds for each unique article, publisher profile, faction page, etc.

Admin Panel

With the new account system in, place, the website needs an admin panel to make doing cool admin stuff easier and more accessible to admins. This admin panel will be strictly available only to admin accounts. This panel will be used for things like managing other accounts, approving new faction uploads, moderating news articles, providing account support, granting or revoking special user permissions, dealing with suspicious or malicious activity, and so on.

The admin panel must look cool because admin stuff = cool. When an admin opens the admin panel, the design of the admin panel alone should make the user feel powerful.

Constellia Subdomain

TFN has partnered with Constellia, another Roblox space game similar to Starscape, in order to list Constellia faction data on the website, separate from Starscape factions. Integrated Forms

All forms, including faction upload forms, will be fully integrated in the website and will no longer rely on Google Forms to function. Users will have to be logged in to submit most forms. Users will be able to upload their faction data to the website, and then that faction will be associated with their account. Only that user will be able to edit data on that faction. Switch to Backend Server

Switch from static pages to using a framework like Express or React to dynamically host and load pages. In essence, this allows quicker dynamic loading of pages like news articles and allows search engines and bots to properly read the content of these pages (rather than seeing "loading headline... loading author... loading content..."). Additionally, this allows for easier and quicker development of such pages.
